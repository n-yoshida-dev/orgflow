# 相談テンプレート

このテンプレートは、毎回すべての項目を埋めるためのものではない。
必要な項目だけを短く書けばよく、未記入の項目があってもよい。
迷った場合は、「今日のテーマ / 目的 / 自分の理解 / 詰まっている点 / 欲しい支援レベル」だけでも十分である。

---

## 1. 今日のテーマ

例: DB設計 / API設計 / Spring Boot実装 / テスト設計 / 認証・認可 / Docker / GitHub Actions / README・ADR / 概念理解

## 2. 今やっていること

今どの作業をしているかを簡潔に書く。

## 3. 目的

この相談で何を前に進めたいかを書く。

## 4. 背景 / 前提

この相談に必要な前提を書く。

## 5. 自分の理解

現時点で自分がどう理解しているかを書く。
「合っている自信はないがこう考えている」でよい。
ここを省くと、AIが完成形を出しやすくなる。

## 6. 自分の案

自分なりの案を書く。
設計相談なら候補A/B、実装相談なら処理の流れ、概念相談なら自分の解釈を書く。

## 7. 詰まっている点

どこで止まっているかを具体的に書く。

## 8. ここまでに考えたこと / 試したこと

何も試していないなら「まだ試していない」でよい。

## 9. 欲しい支援レベル

- レベル1: 問い返しだけ
- レベル2: ヒント中心
- レベル3: 骨子・観点整理まで
- レベル4: 部分例まで
- レベル5: 完全解答OK

指定がない場合、AIはレベル2として扱う。

## 10. 完成形の扱い

- 完成形は出さないでほしい
- まだ完成形は不要。部分例まで
- 最後に比較用の完成例を出してよい
- 今回は完成例を出してよい

## 11. レビューしてほしい観点

必要なものだけ選ぶ / 追記する。
例: 要件理解 / ドメイン理解 / DB正規化・制約 / API責務 / HTTP設計 / 認証・認可 / 権限制御 / 監査ログ / 命名 / 責務分割 / 例外処理 / テスト観点 / 可読性 / 将来拡張性 / SaaSらしさ / README・ADRに残すべき論点

## 12. 回答の希望形式

必要な場合だけ指定する。指定がない場合、AIは「結論 → 主要論点 → 次の1手」の最小構成で返す。

## 13. 添付物 / 対象

コード、SQL、API案、ER図案、README草案などを貼る。長い場合は「どこを見てほしいか」も書く。

## 14. 最後にしてほしいこと

締めで欲しいものを1つだけ書く。11や12と重複する場合は省略してよい。
