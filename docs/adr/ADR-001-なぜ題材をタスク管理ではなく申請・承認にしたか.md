# ADR-001: 題材として申請・承認アプリを採用する

- Status: Accepted
- Date: 2026-03-19

## 背景

- Phase1で、REST API中心の業務アプリとして成立する設計を扱う必要があった
- 単純なCRUD（Todo等）では、以下の論点を自然に含めにくい
  - 承認フロー
  - 組織ごとのデータ分離
  - RBAC
  - 監査ログ
- 転職用ポートフォリオとして、業務ルールを設計に落とし込める題材が必要だった

## 採用した案

申請・承認フローを持つ業務アプリを題材として採用する

## 採用理由

- 業務フロー（申請→承認→監査）を自然に扱える
- organization / role / request / approval / audit_log といった設計要素を分離できる
- 単なるCRUDではなく、設計判断を説明できる成果物になる
- 将来的にSPA・通知・監査UIなどへ拡張しやすい

## 不採用案

- 案A: Todoアプリ
  状態遷移・権限・監査といった論点が弱い
- 案B: タスク管理アプリ
  承認・監査を自然に組み込むのが難しい

## 受け入れる制約

- 見た目の分かりやすさや派手さは弱くなる
- 業務理解が前提となるため、初見の理解コストが上がる

## 見直す条件

- UI訴求を強める必要が出た場合（ダッシュボードや通知機能追加）
- ただし題材自体は変更しない
