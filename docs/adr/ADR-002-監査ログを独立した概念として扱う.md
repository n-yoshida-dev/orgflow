# ADR-002: 監査ログを独立した概念として扱う

- Status: Accepted
- Date: 2026-03-19

## 背景

- 承認履歴だけでなく、ユーザー操作全体を後から追跡したい
- approvalは承認判断の記録であり、監査記録とは役割が異なる
- 「誰が・いつ・何をしたか」を横断的に追跡する必要がある

## 採用した案

監査ログ（audit_log）を独立した概念として設計する

## 採用理由

- 承認以外の操作（ログイン・設定変更など）も記録できる
- 全操作を統一フォーマットで追跡できる
- SaaSらしい監査要件を最初から設計に組み込める

## 不採用案

- 案A: アプリケーションログのみで対応
  業務監査としては不十分
- 案B: approvalで履歴を兼用
  承認以外の操作を表現できない

## 受け入れる制約

- 設計対象が増える
- 監査対象範囲の定義が必要になる

## 見直す条件

- イベント駆動設計を導入する場合
- audit_logとイベント履歴の役割を再整理する必要が出た場合
