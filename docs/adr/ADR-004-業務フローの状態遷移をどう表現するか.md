# ADR-004: 業務フローの状態遷移は明示的な状態モデルで表現する

- Status: Accepted
- Date: 2026-03-19

## 背景

- 申請・承認フローでは状態遷移が中心的な概念になる
- ステータスを曖昧に扱うと、以下の問題が発生する
  - 不正な状態遷移
  - 承認ロジックの分散
  - バグの温床

## 採用した案

状態遷移を明示的なステータスと遷移ルールで表現する

## 採用理由

- 許可される遷移を制御できる
- ビジネスロジックを一箇所に集約できる
- READMEやADRで説明可能な設計になる

## 不採用案

- 案A: ステータスを自由に更新  
  不正遷移が発生しやすい
- 案B: 暗黙的なロジックで管理  
  可読性・保守性が低い

## 受け入れる制約

- 実装がやや冗長になる
- 状態定義・遷移定義の設計コストがかかる

## 見直す条件

- ワークフローエンジンを導入する場合
- 状態遷移が複雑化し、コード管理が限界を超えた場合
